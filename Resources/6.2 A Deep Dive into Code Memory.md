# 🎯 **Cloud Code 記憶（Cloud Code Memory）**

我們會了解：

* 記憶的結構如何設計
* 如何載入
* 最佳實踐
* 如何在團隊與專案中有效使用

## 🧠 什麼是 Cloud Code Memory？

Cloud Code Memory 使用 **Markdown 檔案** 來跨 Session 保存上下文。

它會儲存以下內容：

* 編碼風格
* 命名規範
* 架構設計
* 工具設定

這種創新方式讓 Cloud Code 能在整個專案中記住你的偏好設定，並提升工作流程效率。

你可以把 Cloud Code 的記憶想像成一組「巢狀上下文」，每一層都扮演不同角色，用來調整回應與行為以符合你的實際需求。

## 🎁 Cloud Code Memory 的三大好處

* **三大好處**

### ✅ 1. 消除重複輸入（Eliminate Repetition）

* 不需要在每個 Session 重複輸入相同指示
* 節省大量時間與精力
* 對個人與團隊都非常有效

### ✅ 2. 確保一致性（Ensure Consistency）

* 記憶檔案確保整個專案：

  * 命名一致
  * 架構一致
  * 風格一致
* 有助於團隊協作與程式品質

### ✅ 3. 提升生產力（Boost Productivity）

* 減少頻繁切換上下文的成本
* 專注於實際解題與設計

## 🗂 Cloud Code Memory 的階層架構

Cloud Code 以 **階層式結構（Hierarchical Structure）**

管理記憶，確保最相關資訊優先載入。

### 🏢 1. 企業記憶（Enterprise Memory）

* 由 DevOps 團隊管理
* 存放於企業指定位置
* 強制公司級：

  * 開發標準
  * 最佳實踐
  * 安全規範

### 📦 2. 專案記憶（Project Memory）

* 位於專案根目錄
* 使用 Claude Markdown 檔案
* 包含：

  * 專案規範
  * 架構設計
  * 共用設定
* 可透過 Git 與團隊共享

### 👤 3. 使用者記憶（User Memory）

* 儲存在使用者個人層級
* 適用於所有專案
* 包含：

  * 個人編碼習慣
  * 工具偏好設定

### 💬 4. 即時對話記憶（Conversational Memory）

* 來自你與 Claude 的當前對話內容
* 僅存在於 Session 期間

## 🔍 Claude 如何載入記憶？

當 Claude Code 啟動時：

1. 會從當前目錄向上遞迴掃描至根目錄
2. 依序讀取所有找到的 Claude Markdown 檔案
3. 越上層的檔案優先載入，建立基礎上下文

這種自動載入機制確保所有相關記憶被納入工作上下文。

### 📥 模組化記憶匯入（Memory Import）

你可以在 Claude Markdown 檔案中使用：

```md
add path: ./other-memory.md
```

來匯入其他記憶檔案。

特性：

* 🔁 最多支援 5 層遞迴匯入
* 🧩 允許模組化記憶設計
* 🔒 控制複雜度

## 🧭 記憶管理最佳實踐

### ✍️ 內容撰寫原則

#### ✔ 明確具體（Be Specific）

避免模糊指示。

❌ 不好：

> Format code properly

✅ 好：

> Python 檔案請使用 2 個空白縮排

---

#### ✔ 結構化組織（Structured Organization）

* 使用 Markdown 標題
* 使用條列清單
* 依主題分類

#### ✔ 精簡（Keep It Lean）

* 記憶檔會佔用 Context Window
* 保持精簡可提升效能

## 🔧 維護策略

### 🔄 定期更新（Periodic Review）

隨專案演進更新規範與偏好。

### 🔍 定期稽核（Regular Audits）

比對實際程式碼是否符合記憶內容。

### 📏 大小限制（Size Limit）

* 建議單一記憶檔 **不超過 500 行**
* 確保快速載入與處理效率

✅ 團隊一致性
